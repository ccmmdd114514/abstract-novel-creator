# 子技能3：平台标准正文生成

## 适用场景
剧情方案确定后，按照平台要求的风格和节奏生成章节正文初稿。

## 输入格式

```
【本章剧情方案】
（来自子技能2的输出）

【写作风格参考】
（平台要求的风格说明，或前文章节的风格样本）
```

## 输出格式

```
【本章正文】
（按网文平台格式生成的正文内容，纯文本）

【正文统计】
- 总字数：[X]
- 场景数：[X]
- 出场人物数：[X]
- 章节结尾类型：[悬念/转折/高潮/平稳过渡/笑点收尾]
```

## 执行步骤

1. 按照剧情方案的场景拆解，逐场景生成正文
2. 保持语言风格与平台要求一致（节奏明快、网文化表达）
3. 嵌入抽象系统的互动和抽象元素
4. 科技元素自然融入剧情，不割裂
5. 控制章节节奏，在适当位置设置悬念或笑点
6. 章节结尾确保有钩子，让读者想往下翻
7. 输出完整正文及统计信息

## 注意事项

- 注意章节结尾的吸引力——断章技巧是网文命脉
- 保持和意味和抽象系统的对话风格一致性
- 和意味说话带程序员抽象味，抽象系统已读乱回
- 对话占比不低于全文30%，避免大段纯叙述
- 每章控制在平台推荐的字数范围内（通常2000-4000字）
