# 主技能：平台小说章节全流程创作

## 适用场景
每章创作的全流程调度，从资料整理到最终备份，协调所有子技能完成从资料到正文的完整创作链路。

## 7步循环 × 子技能调用对照表

| 步骤 | 操作 | 调用的子技能 | 产出 |
|------|------|-------------|------|
| 1 | 资料整理 | 资料整理与前文梳理 | 前文状态报告 |
| 2 | 情节推演 | 单章剧情推演 | 本章剧情方案 |
| 3 | 正文生成 | 平台标准正文生成 | 正文初稿 |
| 4 | 合规审查 | 合规与一致性审查 | 审查报告 |
| 5 | 内容优化 | 内容优化与打磨 | 优化后正文 |
| 6 | 世界线更新 | 故事线与人物线更新 | 更新后的追踪文档 |
| 7 | 自动备份 | 自动版本备份 | 版本备份 |

## 输入格式

```
【用户指令】
（用户对本章剧情的描述和要求）

【当前项目状态】
- 当前章节号
- 已发布章节数
- 最新故事线状态文件路径
```

## 输出格式

```
【创作流程报告】
✅ [步骤1] 资料整理完成  → 输出：[前文状态报告路径]
✅ [步骤2] 情节推演完成  → 输出：[剧情方案路径]
✅ [步骤3] 正文生成完成  → 输出：[正文初稿路径]
✅ [步骤4] 合规审查完成  → 输出：[审查报告路径]
✅ [步骤5] 内容优化完成  → 输出：[优化后正文路径]
✅ [步骤6] 世界线更新完成 → 输出：[更新报告路径]
✅ [步骤7] 自动备份完成  → 输出：[备份路径]

【最终交付物】
- 本章正文：[路径]
- 更新后故事线：[路径]
- 版本备份：[路径]
```

## 执行步骤

> **执行规范：** 每一步执行时必须先输出步骤进度标识，格式为 `【步骤X/7 — 步骤名称】`，让用户清楚当前所处环节。

### 步骤1 — 资料整理
**调用技能：** 资料整理与前文梳理
**动作：** 读取人物状态、前文梗概、未回收伏笔
**产出：** 结构化前文状态报告（含剧情进度、人物状态、活跃伏笔、当前星球规则）

### 步骤2 — 情节推演
**调用技能：** 单章剧情推演
**动作：** 规划本章3-5个核心事件，确定冲突和节奏
**输入依赖：** 步骤1产出的前文状态报告 + 用户剧情指令
**产出：** 本章剧情方案（场景拆解、冲突设计、抽象系统介入点、伏笔安排、节奏曲线）

### 步骤3 — 正文生成
**调用技能：** 平台标准正文生成
**动作：** 按情节大纲或用户输入剧情生成正文初稿
**输入依赖：** 步骤2产出的本章剧情方案
**产出：** 本章正文初稿（纯文本，待审查优化）

### 步骤4 — 合规审查
**调用技能：** 合规与一致性审查
**动作：** 检查 OOC（人设崩塌）、前后文矛盾、平台违规内容（政治/色情/暴力）
**输入依赖：** 步骤3产出的正文初稿
**产出：** 审查报告（通过/不通过 + 问题清单 + 修改建议）
**分支逻辑：** 如审查不通过（存在P0问题），返回步骤2-3修正后重新审查；通过则继续

### 步骤5 — 内容优化
**调用技能：** 内容优化与打磨
**动作：** 优化对话自然度、调整叙事节奏、增强章节悬念
**输入依赖：** 步骤3正文初稿 + 步骤4审查报告（如有问题标注）
**产出：** 优化后正文
**说明：** 如步骤4审查无问题，本步骤仍可进行润色优化；如有问题则针对性修改

### 步骤6 — 世界线更新
**调用技能：** 故事线与人物线更新
**动作：** 更新人物状态、伏笔清单（新埋设/已回收）、追加本章梗概至故事线
**输入依赖：** 步骤5优化后正文
**产出：** 更新后的追踪文档 + 更新报告

### 步骤7 — 自动备份
**调用技能：** 自动版本备份
**动作：** 将本章正文和元数据保存至版本备份目录
**输入依赖：** 步骤5优化后正文 + 步骤2剧情方案 + 步骤6更新报告
**产出：** 版本备份（按版本号归档）+ 备份报告

## 注意事项

- 7个步骤严格按顺序执行，前一步的产出作为下一步的输入
- P0级别问题（人设崩塌/违规内容）必须修复后才能进入下一环节
- 如中间步骤失败，记录错误并终止流程，不进入下一步
- 步骤5在审查无问题时仍建议执行润色优化（最低成本提升质量）
- 每一步的产出文件统一存放至对应目录
