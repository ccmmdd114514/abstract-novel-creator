# 子技能1：资料整理与前文梳理

## 适用场景
每章创作前，快速了解当前故事状态——前文剧情进度、人物状态、未回收伏笔、活跃设定项。

## 输入格式

```
【项目目录】
（项目根目录路径）

【当前章节】
（计划创作的章节号）
```

## 输出格式

```
【前文状态报告】
📖 剧情进度：
  - 当前主线进度：[已完成/进行中]
  - 当前星球：[星球名称/世界观]
  - 上一章概要：[简述]

👤 人物状态：
  - 和意味：[当前状态/位置/心态]
  - 抽象系统：[当前状态]
  - 本章登场角色：[角色名+状态]

🔗 未回收伏笔：
  - [伏笔1] → 已埋设章节：[章号]
  - [伏笔2] → 已埋设章节：[章号]

📋 活跃设定项：
  - 当前星球规则：[简述]
  - 可用科技元素：[列表]
```

## 执行步骤

1. 读取当前故事线状态文件，获取最新剧情进度
2. 扫描最新一章的正文，提取关键剧情进展
3. 读取人物设定文档，标记各人物当前状态
4. 读取伏笔追踪表，列出所有未回收伏笔
5. 整理当前星球/世界的活跃设定项
6. 输出结构化前文状态报告

## 注意事项

- 不修改任何现有文件，只读操作
- 只输出状态报告，不生成新内容
- 伏笔追踪需区分「已埋设」「已回收」「已过期」三种状态
