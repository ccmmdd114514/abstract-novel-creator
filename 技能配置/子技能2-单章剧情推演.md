# 子技能2：单章剧情推演

## 适用场景
资料整理完成后，基于全书大纲和用户指令，生成具体章节的剧情推演方案——场景拆解、冲突设计、节奏规划。

## 输入格式

```
【前文状态报告】
（来自子技能1的输出）

【用户剧情指令】
（用户对本章剧情的描述和要求，包括抽象系统介入点）

【全书大纲】（可选）
（如需要引用大纲内容辅助决策）
```

## 输出格式

```
【本章剧情方案】
🎯 章节定位：
  - 章节号：[X]
  - 本章核心作用：[推进主线/塑造人物/铺垫伏笔/过渡衔接/高潮爆发]

📝 剧情概要：
  （200字以内的本章剧情梗概）

🎬 场景拆解：
  场景1：[场景名]
    - 地点：
    - 登场人物：
    - 核心冲突：
    - 科技元素融合：
    - 预计字数：[字]
  
  场景2：[场景名]
    - ...

💡 抽象系统介入设计：
  - 系统改剧本点：[位置+改了啥+效果]
  - 抽象效果/笑点：[位置+内容]

🔗 伏笔安排：
  - 新埋设伏笔：[内容+预期回收章号]
  - 回收伏笔：[内容+关联章号]

⚡ 章节节奏曲线：
  - 起：[场景1内容]
  - 承：[场景2内容]
  - 转：[场景3转折点]
  - 合：[章节结尾收束/悬念]
```

## 执行步骤

1. 读取前文状态报告，确认当前故事所处阶段
2. 结合全书大纲和用户剧情指令，规划本章在全书的定位
3. 拆解本章场景，设计每个场景的冲突和节奏
4. 设计抽象系统的介入点和科技融合点
5. 安排伏笔的埋设或回收
6. 规划章节节奏曲线（起承转合）
7. 输出结构化剧情方案

## 注意事项

- 必须符合全书大纲走向，不得偏离主线
- 人设不能崩——和意味就是和意味，抽象系统就是抽象系统
- 节奏控制：避免信息密度过高或过低
- 章节结尾必须留钩子（悬念/转折/笑点），确保读者想往下看
- 每颗星球（网文世界）的元素必须体现50%科技+50%随机/抽象的融合
